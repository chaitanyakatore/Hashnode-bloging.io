If developers aren't commenting when writing code, even the cleanest code can be a challenge for debugging.Debugging is an important part of determining why our program is not working properly.

Debugging strategies
  • Static analysis- The developer test the code without executing the program.
  • Tracing- The developer watches live print statements and monitors flow.
  • Remote debugging- The developer's debugger runs on a different system than the program that is being debugged.
  • Post-mortem debugging- The developer only stops to debug the program if it experiences fatal exceptions.
